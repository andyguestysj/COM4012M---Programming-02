### Notes on Changes

- **Lecture 4 (new):** Covers what AI coding assistants are, effective prompting for web development tasks, critical evaluation of AI output, academic integrity, and limitations specific to web dev (outdated APIs, accessibility gaps, security anti-patterns). Replaces the standalone JavaScript Basics lecture.
- **Lecture 5 (merged):** JavaScript fundamentals (variables, types, functions, the Python comparison) are condensed into the opening section of what was the DOM lecture. Students with Python backgrounds pick up JS syntax quickly; a full standalone lecture is no longer needed.
- **Lecture 10 (reduced):** ASP.NET is retained as a conceptual contrast to PHP — students understand that server-side development is not limited to one language — but is no longer treated as an assessable technology. The lab moves from scaffolding a full ASP.NET project to a comparative discussion exercise.

## Summary of Changes

| Area | What Changed | Rationale |
|------|-------------|-----------|
| **New lecture** | Dedicated AI Tools lecture at Week 4 | Students need a framework for using AI before they encounter complex JS and PHP content |
| **Merged lecture** | JS Basics folded into DOM lecture | Python-background students do not need a full standalone lecture on JS syntax |
| **Reduced lecture** | ASP.NET reduced to conceptual overview | ASP.NET was never assessed; reducing it creates space without losing the breadth signal |
| **New hand-in** | AI Development Log (300–500 words) | Makes AI use transparent, assessable, and pedagogically meaningful |
| **Weighting shift** | Technology +5%, Standards −5% | Reflects that validation is increasingly automated; critical technology use is the differentiator |
| **AI policy** | Restrictive → Professional | The current policy is unenforceable and inconsistent with teaching AI as a topic; the new framing is honest and industry-aligned |
